# Impeccable Design Context — Open Managed Agents Console

## Design Context

### Users
Developers and platform engineers who deploy, configure, and debug autonomous AI agents. They interact with this console to create agents, manage sessions, set up sandbox environments, and handle credential vaults. They expect a tool that feels as sharp as their CLI but visual enough to monitor complex agent lifecycles at a glance.

### Brand Personality
**Warm, Technical, Playful**

The brand identity centers on the `[horse]` logo — a horse head in profile sketched with minimal strokes (ear, eye, muzzle), framed by monospace brackets in JetBrains Mono. Three layers of meaning: "Open **Ma**naged Agents" abbreviates to **openma**, where "ma" (马) is Chinese for horse; the platform's core abstraction is the **Harness** — literally horse tack, reins for controlling a mount; and the product itself harnesses AI agents the way a rider harnesses a horse. The logo is all three made visual. The coral brand color (`#FF6B50`) reinforces warmth and approachability without sacrificing professionalism.

Emotional goals: **confidence** when managing agents, **clarity** when debugging sessions, and a subtle **delight** from a UI that feels crafted rather than generated.

### Aesthetic Direction

**Visual tone**: Warm technical — clean structure with personality through color and typography.

**References**:
- Anthropic Console: structural patterns (sidebar groups, list page templates, modal forms)
- Vercel: information density done right, developer-first
- Linear: smooth interactions, purposeful motion

**Anti-references**:
- Generic admin dashboards (Ant Design defaults, Bootstrap grids)
- Glassmorphism, gradient text, decorative blur — these are AI design tells
- Cold/sterile enterprise UIs with no personality

**Theme**: Light + Dark mode, both required. The brand color must work well in both contexts.

### Color System

|-------|-------|------|---------|
| `--brand` | `#FF6B50` | `#FF6B50` | Primary actions, active states, logo |
| `--brand-hover` | `#E85A40` | `#FF8068` | Hover state for brand elements |

| `--bg-surface` | `#F5F5F5` | `#222222` | Cards, code blocks, table rows |
| `--fg` | `#171717` | `#EDEDED` | Primary text |
| `--fg-muted` | `#737373` | `#999999` | Secondary text, descriptions |
| `--fg-subtle` | `#767676` | `#9B9B9B` | Tertiary text — kicker labels, dates, breadcrumbs (passes AA-normal) |
| `--border` | `#E5E5E5` | `#333333` | Dividers, table borders |
| `--border-strong` | `#D4D4D4` | `#444444` | Focused input borders |

**Body-link treatment.** Coral brand (`#FF6B50`) fails AA-normal text contrast (~3.4:1) on white. Rather than introduce a second "darkened coral" token (which would split the brand identity), body links inside prose paragraphs/lists/cells inherit the surrounding text color and use coral on the **underline** instead — `text-decoration-color: var(--brand)`, `text-decoration-thickness: 2px`. Decoration colors aren't subject to WCAG 1.4.3, so the brand stays visible in body text while the readable text color stays at 14:1+. Hover transitions the full text color to coral as an interaction signal. Header / footer / breadcrumb links inherit the surrounding fg color (no underline); brand color is reserved for fills (buttons, badges) and the underline / hover signal here.

### Typography

| Role | Font | Weight | Size |
|------|------|--------|------|
| Display / headings | Source Serif 4 | 600 | 24–32px |
| Body / UI | DM Sans | 400 / 500 | 14–16px |
| Code / IDs | JetBrains Mono | 400 | 13–14px |
| Sidebar labels | DM Sans | 500 | 13px |

Scale ratio: 1.25 (major third). Base: 14px.

### Spacing & Layout

- Base unit: 4px
- Semantic tokens: `--space-xs: 4px`, `--space-sm: 8px`, `--space-md: 16px`, `--space-lg: 24px`, `--space-xl: 32px`, `--space-2xl: 48px`
- Sidebar width: 240px (collapsible)
- Content max-width: none (full bleed within main area)
- Border radius: `4px` (inputs/buttons), `8px` (cards/modals), `12px` (sidebar/main panels)
- Shadows: minimal — `0 1px 2px rgba(0,0,0,0.05)` for light, `0 1px 2px rgba(0,0,0,0.3)` for dark

### Design Principles

1. **Brand color is the hero** — `#FF6B50` is the single accent. No competing colors for primary actions. Every interactive element traces back to the coral.

2. **Structure from Anthropic, soul from the horse** — Adopt Anthropic Console's layout patterns (grouped sidebar, list+filter+table pages, centered modals) but inject warmth through the brand color, serif headings, and considered typography pairing.

3. **Light and Dark are first-class citizens** — Neither mode is an afterthought. Design tokens must resolve correctly in both. Test in both.

4. **Information over decoration** — Developers need data density. Use whitespace for hierarchy, not aesthetics. Every pixel should earn its place.

5. **Monospace is identity** — Agent IDs, session IDs, code blocks, and the logo brackets all share JetBrains Mono. This typographic thread ties the UI to its developer audience and the brand mark `[ ]`.
